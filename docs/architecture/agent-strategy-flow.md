## 0. 重要说明：两种分析模式

系统有两种不同的分析模式，取决于配置：

| 模式 | 触发条件 | 流程特点 |
|------|----------|----------|
| **传统 Pipeline 模式** | `AGENT_MODE=false` 且未配置 `AGENT_SKILLS` | 单次 LLM 调用，结构化 Prompt |
| **Agent 模式** | `AGENT_MODE=true` 或配置了 `AGENT_SKILLS` | 多轮对话，Agent 自主决策调用工具 |

### 如何查看当前配置

```bash
# 查看环境变量
echo $AGENT_MODE
echo $AGENT_SKILLS
```

### 本次测试结果

本次测试执行的是 **Agent 模式**，因为配置了：
```
AGENT_SKILLS=bull_trend,ma_golden_cross,volume_breakout,shrink_pullback
```

## 2. 模式一：传统 Pipeline 模式（单次 LLM 调用）

当 `AGENT_MODE=false` 且未配置 `AGENT_SKILLS` 时使用此模式。

### 流程步骤

```
Step 4 结束
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 多维度情报搜索 (SearchService)                    │
│  引擎: Bocha > Tavily > Brave > MiniMax (故障转移)        │
│  维度: 最新消息、风险排查、业绩预期                         │
│  输出: news_context (格式化情报报告)                       │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 6: 并行数据收集器 (ThreadPoolExecutor, max_workers=3)│
│                                                             │
│  ┌──────────────────┐  ┌──────────────────┐                │
│  │ FundFlowCollector│  │ NewsPreprocessor │                │
│  │ 北向资金         │  │ LLM语义分类      │                │
│  │ 主力净流入       │  │ 利好/利空判断    │                │
│  │ 龙虎榜           │  │ 时效性加权       │                │
│  └──────────────────┘  └──────────────────┘                │
│                                                             │
│  ┌──────────────────┐  ┌──────────────────┐                │
│  │ PeerComparison  │  │ TechnicalPattern │                │
│  │ Collector       │  │ Labeler         │                │
│  │ 板块对比         │  │ 多头排列         │                │
│  │ 竞品数据         │  │ MACD金叉        │                │
│  │ 相对估值         │  │ RSI超买/超卖    │                │
│  └──────────────────┘  └──────────────────┘                │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 7: 上下文增强 (_enhance_context)                    │
│  注入: 实时行情、筹码、趋势、资金流向、分类新闻、同行对比   │
│  输出: enhanced_context (完整分析上下文)                   │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 8: AI 分析 (GeminiAnalyzer.analyze)                │
│  输入: enhanced_context + news_context                     │
│  特点: 单次 LLM 调用，结构化 Prompt                        │
│  输出: AnalysisResult (评分、建议、风险提示)              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 9: 结果处理                                          │
│  - 保存 AnalysisHistory 到数据库                           │
│  - 生成报告文件 (markdown)                                │

└─────────────────────────────────────────────────────────────┘
```

### 关键特点

- **单次 LLM 调用**：一次 prompt 包含所有上下文，一次返回结果
- **结构化 Prompt**：所有数据预先整理成结构化格式
- **并行数据收集**：FundFlow/NewsPreprocessor/PeerComparison/TechnicalPattern 并行获取
- **速度较快**：一般 30-60 秒完成

## 3. 模式二：Agent 模式（多轮对话）

当 `AGENT_MODE=true` 或配置了 `AGENT_SKILLS` 时使用此模式。

### 流程步骤

```
Step 4 趋势分析 结束
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 5: Agent 模式检测                                    │
│  判断: agent_mode=true 或 配置了 agent_skills             │
│  技能: bull_trend, ma_golden_cross, volume_breakout 等    │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 6: Agent 执行循环 (Agent Loop)                       │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ Agent Step 1: 理解任务，制定计划                      │  │
│  │   └→ 调用工具: get_daily_history (获取历史数据)     │  │
│  ├─────────────────────────────────────────────────────┤  │
│  │ Agent Step 2: 分析趋势                               │  │
│  │   └→ 调用工具: analyze_trend (趋势分析)             │  │
│  ├─────────────────────────────────────────────────────┤  │
│  │ Agent Step 3: 搜索新闻                               │  │
│  │   └→ 调用工具: search_stock_news (搜索情报)         │  │
│  ├─────────────────────────────────────────────────────┤  │
│  │ Agent Step 4: 综合分析                               │  │
│  │   └→ 调用工具: analyze_stock (综合分析)             │  │
│  ├─────────────────────────────────────────────────────┤  │
│  │ ... (最多 agent_max_steps=10 轮)                    │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                             │
│  Agent 自主决策每轮调用哪个工具，可根据分析结果动态调整     │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 7: Agent 完成                                         │
│  输出: 分析结果 + 保存新闻情报                              │
│  耗时: 一般 60-90 秒 (多轮对话)                           │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 8: 结果处理                                          │
│  - 保存 AnalysisHistory 到数据库                           │
│  - 生成报告文件 (markdown)                                │
│  - 发送通知 (邮件/微信/飞书)                               │
└─────────────────────────────────────────────────────────────┘
```

### 关键特点

- **多轮对话**：Agent 可以多轮思考和调用工具
- **自主决策**：Agent 根据每轮结果决定下一步行动
- **灵活调用**：可以调用各种预定义的 Tool（如 get_daily_history, analyze_trend 等）
- **策略技能**：支持配置 skills（如 bull_trend 牛市趋势、ma_golden_cross 金叉策略等）
- **速度较慢**：一般 60-90 秒，但分析更深入

### 可用 Tools (src/agent/tools/)

| Tool 名称 | 功能 |
|-----------|------|
| `get_daily_history` | 获取股票历史K线数据 |
| `get_realtime_quote` | 获取实时行情 |
| `analyze_trend` | 技术趋势分析 |
| `search_stock_news` | 搜索股票新闻 |
| `analyze_stock` | 综合分析股票 |
| `get_chip_distribution` | 获取筹码分布 |
| `get_fundamental` | 获取基本面数据 |

## 8. 快速排查指南

### 分析未触发
1. 检查是否有 `[盘后缓存] 命中` 日志 → 有则正常跳过
2. 检查 `force_update` 参数是否传递正确

### 数据获取失败
1. 检查数据源是否配置 (Efinance/Akshare API)
2. 检查网络连接
3. 查看具体错误日志

### LLM 分析失败
1. 检查 API Key 配置
2. 检查模型可用性
3. 查看 `analyze` 方法的错误日志
